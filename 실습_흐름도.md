## 🅑 단계: Neo4j 로컬 그래프 시각화

> 설치 완료 후 아래 Cypher 쿼리로 직접 탐색

### 데이터 import (Neo4j Browser에서 실행)

```cypher
// 1. nodes.jsonl에서 노드 생성 (Neo4j Desktop APOC 플러그인 사용)
CALL apoc.load.jsonArray('file:///nodes.jsonl')
YIELD value
CREATE (n:OntologyNode)
SET n = value.properties, n.id = value.id,
    n.label = value.label, n.space = value.space

// 2. 관계 생성
CALL apoc.load.jsonArray('file:///edges.jsonl')
YIELD value
MATCH (a {id: value.source}), (b {id: value.target})
CREATE (a)-[:RELATION {type: value.relation}]->(b)
```

### 연습용 Cypher 쿼리

```cypher
-- Python과 연결된 모든 노드
MATCH (n)-[r]-(m) WHERE n.id = 'subj_python'
RETURN n, r, m

-- Claim과 근거(Evidence) 관계
MATCH (c {space:'Claim'})-[r]->(e {space:'Evidence'})
RETURN c.label, type(r), e.label

-- ML 관련 전체 서브그래프
MATCH path = (n)-[*1..2]-(m)
WHERE n.id = 'con_ml'
RETURN path

-- Intensity 점수 높은 순 정렬
MATCH (n {space:'Intensity'})
RETURN n.label, n.score
ORDER BY n.score DESC
```

## 🔄 내 도메인으로 확장하는 방법

```
1. 주제 선정         → 업무/관심사 1개 (너무 넓지 않게)
2. 노드 설계         → 9 시맨틱 스페이스 중 해당하는 것 매핑
3. JSONL 작성        → nodes.jsonl / edges.jsonl
4. documents 작성    → 원본 텍스트 → 문장 단위 청크
5. ZIP 패키징        → cloud/ + graph/ 구조 유지
6. OpenCrab 업로드   → Pack 생성 → MCP 연결
7. GitHub push       → git add . → git commit → git push
```

| 프로젝트 도메인 지식 | 개념/정책/이해관계자 | 적용관계/규칙 |

| 회사 도메인 (비공개) | 프로세스/규정/담당자 | 승인흐름/관계 |

## ⚠️ 핵심 주의사항

```
✅ 청크는 반드시 문장 단위로 (단어/쉼표 단위 금지)
✅ 노드에 id 필드 필수
✅ 엣지에 source/target 필수
✅ ZIP 최상단에 cloud/, graph/ 폴더가 위치해야 함
✅ 팩 1개 = 역할 1개 (너무 넓은 주제는 여러 팩으로 분리)
❌ ZIP 엔트리 500개 초과 금지
❌ 단일 파일 5MB 초과 금지 (완화 시 최대 30MB)
```
